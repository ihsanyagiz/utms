## Auth Route Map

| Endpoint | Who uses it | Role selector |
|---|---|---|
| `/auth/applicant/login` | Applicants | No |
| `/auth/applicant/register` | Applicants | No (open self-registration) |
| `/auth/staff/login` | Staff (oidb, dean, ygk, ydyo) | Yes â€” no register link shown |
| `/auth/staff/register` | **Disabled** â€” redirects to staff login | â€” |
| `/auth/admin/login` | Admin only | No â€” role is hardcoded "admin" |

Admin is **not** in `_STAFF_SLUGS` â€” the staff login rejects it. Admin has its own login route, not linked from the public portal. Navigate to it directly.

Staff self-registration is disabled. Staff accounts must be provisioned by an admin (see TODO in `app/AGENTS.md`).

## EXPLICIT CODING RULES

### âś… DO:
1. **Delegate to services or role classes** â€” routes call one method and handle the result
2. **Use `@role_required("slug")`** â€” all non-auth routes protecting a role must use this decorator
3. **Flash static success messages** â€” hardcoded strings, not extracted from result.data
4. **Flash `result.error_msg` for errors** â€” the business layer already formatted it
5. **Redirect after POST** â€” always redirect after successful mutations (PRG pattern)

### âťŚ DO NOT:
1. **Direct database access** â€” never use `db.session`, `ApplicationORM.query`, etc.
2. **Business logic** â€” no form validation, no status checks, no complex conditions
3. **Direct class instantiation in routes** â€” instantiate inside a `_svc()` / `_admin()` helper, not inline
4. **isinstance checks on result.data** â€” the OperationResult contract guarantees the type
5. **Extract messages from result.data** â€” `result.data.get("message")` is wrong; use static flash

## Pattern: Correct Route

```python
@admin_bp.route("/users/change-role", methods=["POST"])
@role_required("admin")
def change_user_role():
    try:
        target_id = int(request.form.get("user_id", 0))
    except ValueError:
        flash("Invalid user ID.", "error")
        return redirect(url_for("dashboard.index"))

    new_role = (request.form.get("new_role") or "").strip()
    result = _admin().change_user_role(target_id, new_role)

    if result.success:
        flash(f"Role updated to '{new_role}'.", "success")   # static message
    else:
        flash(result.error_msg, "error")                     # from OperationResult

    return redirect(url_for("dashboard.index"))
```

## Dashboard Route (`dashboard.py`)

`_get_tab_data(role_slug, tab_slug, page, user_id, email, filter_status, sort_col, sort_dir)` returns the template context for one tab. Add a new branch here when a new tab needs data.

**Filter/sort**: For paginated staff panels, `_get_tab_data` calls `_filter_apps` then `_sort_apps` on the **full** app list before slicing. Filter and sort arrive as query params (`?filter=submitted&sort_col=1&sort_dir=asc`). `tab_content()` reads them from `request.args` and forwards to `_get_tab_data`. Status chip counts are computed on the unfiltered list so totals stay accurate regardless of active filter.

**`_filter_apps`** supports comma-separated status values (e.g. `"forwarded_to_ydyo,forwarded_to_dean"`) for multi-status chips.

**`_sort_apps`** uses a `(bucket, numeric, string)` tuple key â€” integers sort numerically, datetimes by ISO string, strings case-insensitively, and `None` values always sort last.

Role display name lookup uses `_ROLE_DISPLAY` (a local dict) rather than `ROLE_DEFINITIONS`, so admin shows "Administrator" without being added to the public portal's role list.

## Anti-Pattern: What NOT to Do

```python
# âťŚ WRONG â€” direct class use with inline instantiation
applicant = Applicant(session["user_id"], session["user_email"], "applicant")
form_data = applicant.parse_form_data(request.form)   # form parsing in route

# âťŚ WRONG â€” direct database
from app.models import ApplicationORM
app = ApplicationORM.query.get(id)
app.status = "new_status"
db.session.commit()

# âťŚ WRONG â€” extracting message from result.data
if result.success:
    flash(result.data.get("message"), "success")   # ApplicationORM has no .get()
```
