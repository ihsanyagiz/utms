**Purpose**: SQLAlchemy ORM model definitions ‚Äî data shape only, no business logic

## Naming Rule ‚Äî ORM Suffix + Explicit `__tablename__`

All ORM model classes carry the `ORM` suffix to distinguish them from domain entities in `classes/entities/`.

**Always declare `__tablename__` explicitly.** SQLAlchemy derives the table name from the class name in lowercase snake-case. When the class name changes (e.g. to add the `ORM` suffix), the derived name would change too (`UserORM` ‚Üí `user_orm`), which would break existing databases. Explicit `__tablename__` prevents this.

```python
class UserORM(db.Model):
    __tablename__ = "user"          # REQUIRED ‚Äî explicit, matches existing table
    ...

class ApplicationORM(db.Model):
    __tablename__ = "application"   # REQUIRED
    ...

class ApplicationDocumentORM(db.Model):
    __tablename__ = "application_document"  # REQUIRED
    ...
```

**Any new ORM model must follow the same pattern: `SomethingORM` class + `__tablename__` explicitly set.**

## Audit Columns ‚Äî Do NOT Set Manually

`ApplicationORM` has two audit columns:

```python
last_edited_by_id = db.Column(db.Integer, nullable=True)
last_edited_at    = db.Column(db.DateTime, nullable=True)
```

These are set **automatically** by the `before_flush` SQLAlchemy event listener in `app/__init__.py`. It reads `flask.session["user_id"]` and stamps every dirty or new `ApplicationORM` before each commit.

- ‚ùå Never set `last_edited_by_id` or `last_edited_at` in business code.
- ‚úÖ The listener handles it. If you see manual assignments to these fields, remove them.

`ApplicationORM.parsed_checker_errors` is a **property** (not a column) that returns the JSON as a Python list of dicts, safe to iterate in templates. It returns `[]` on missing or malformed JSON.

## Adding a New Column

Two changes required ‚Äî or existing databases break on startup:

1. **`models/application.py`** ‚Äî add the `db.Column(...)` declaration to `ApplicationORM`.
2. **`app/__init__.py` ‚Üí `_ensure_application_new_columns()`** ‚Äî add the column name and its SQLite type string to the `additions` dict.

The helper runs `ALTER TABLE application ADD COLUMN ‚Ä¶` for any missing column on startup.

## EXPLICIT CODING RULES

### ‚úÖ DO:
1. **Declare `__tablename__`** on every ORM model ‚Äî no exceptions
2. **Suffix class names with `ORM`** to distinguish from domain entities
3. **Keep models data-only** ‚Äî columns, relationships, FK constraints, `set_password` / `check_password` on UserORM
4. **Update `_ensure_application_new_columns()`** when adding columns to ApplicationORM

### ‚ùå DO NOT:
1. **Put business logic in models** ‚Äî no state transition methods, no workflow decisions
2. **Import models in business classes directly** ‚Äî business classes use repositories, not models
3. **Set `last_edited_by_id` or `last_edited_at` manually** ‚Äî the listener owns these
4. **Use legacy class names** (`User`, `Application`, `ApplicationDocument`) ‚Äî always use the `ORM` suffix forms

## Relationships

```python
# UserORM
application = db.relationship("ApplicationORM", back_populates="applicant", uselist=False)

# ApplicationORM
applicant  = db.relationship("UserORM", back_populates="application")
documents  = db.relationship("ApplicationDocumentORM", back_populates="application",
                              cascade="all, delete-orphan")

# ApplicationDocumentORM
application = db.relationship("ApplicationORM", back_populates="documents")
```

Relationship string arguments reference the class name (`"ApplicationORM"`, not the table name `"application"`).
