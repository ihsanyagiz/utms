# Database Directory — Coding Rules

**Location**: `app/database/`  
**Purpose**: Data access layer — repositories and adapters  

## Structure

```
database/
  __init__.py               # Factory functions: get_application_repository(),
                            #   get_user_data_interface(), get_user_repository(),
                            #   get_database_interface()
  application_repository.py # Typed ApplicationORM data access
  adapters.py               # SQLAlchemyDatabaseAdapter + SQLAlchemyUserDataService
  interfaces.py             # ILowLevelDatabase, IHighLevelDatabase, IUserRepository
```

## EXPLICIT CODING RULES

### ✅ DO:
1. **Use ORM class names** — `ApplicationORM`, `ApplicationDocumentORM`, `UserORM` (never legacy bare names)
2. **Return OperationResult** — all repository methods return `OperationResult` with error codes
3. **Use selectinload for relationships** — eager-load related data to avoid N+1 queries
4. **Handle SQLAlchemy exceptions** — convert db errors to `ErrorCode.DB_ERROR`
5. **Centralize queries here** — all model queries belong in repositories, NOT in routes or services

### ❌ DO NOT:
1. **Create new string-based dispatch** — `safe_execute` uses a whitelist; user operations must be added to that whitelist
2. **Put db.session calls in business classes** — only repositories touch `db.session`
3. **Return bare None** — wrap errors in OperationResult, never silently return None
4. **Query models in routes** — move all `ApplicationORM.query.*` calls to `ApplicationRepository`

## Pattern: Correct Repository Method

```python
@staticmethod
def find_by_id(application_id: int) -> OperationResult:
    try:
        app = ApplicationORM.query.get(application_id)
        if app is None:
            return OperationResult.error(ErrorCode.NOT_FOUND, "Application not found")
        return OperationResult.ok(data=app)
    except Exception as e:
        return OperationResult.error(ErrorCode.DB_ERROR, str(e))
```

## Anti-Pattern: What NOT to Do

```python
# ❌ WRONG — legacy model name
from ..models import Application    # use ApplicationORM

# ❌ WRONG — query in route
from app.models import ApplicationORM
app = ApplicationORM.query.filter_by(applicant_id=user_id).first()

# ❌ WRONG — silent None
def find_by_id(self, app_id: int) -> ApplicationORM | None:
    return ApplicationORM.query.get(app_id)   # Returns None silently
```

## Available Repositories

**ApplicationRepository** (via `get_application_repository()`)

| Method | Returns |
|---|---|
| `find_by_applicant_id(id)` | `OperationResult[ApplicationORM \| None]` |
| `find_by_id(id)` | `OperationResult[ApplicationORM \| None]` |
| `create(applicant_id, **kwargs)` | `OperationResult[ApplicationORM]` |
| `update(application)` | `OperationResult[ApplicationORM]` |
| `update_status(app_id, status)` | `OperationResult[ApplicationORM]` |
| `commit()` | `OperationResult` |
| `rollback()` | `OperationResult` |
| `get_all()` | `OperationResult[list[ApplicationORM]]` |
| `get_pending_for_dean/oidb/ydyo()` | `OperationResult[list[ApplicationORM]]` — eager-loads `.documents` and `.applicant` |
| `get_intibak_complete_apps()` | `OperationResult[list[ApplicationORM]]` — status=`intibak_complete`, eager-loads `.intibak_table` and `.applicant` |
| `get_by_status(status)` | `OperationResult[list[ApplicationORM]]` — no relationship eager-loading |
| `get_by_status_with_documents(status)` | `OperationResult[list[ApplicationORM]]` — eager-loads `.documents` and `.applicant`; required by DocumentChecker |
| `get_document(doc_id)` | `OperationResult[ApplicationDocumentORM]` |
| `add_document(doc)` | `OperationResult` |
| `add_documents(docs)` | `OperationResult` |
| `delete_document(doc)` | `OperationResult` |
| `delete_documents(app_id)` | `OperationResult` |

**IntibakRepository** (via `get_intibak_repository()`)

| Method | Returns |
|---|---|
| `get_or_create_table(app_id)` | `OperationResult[IntibakTableORM]` |
| `add_course(table_id, side, data)` | `OperationResult[IntibakCourseORM]` |

**UserRepository** (via `get_user_repository()` or `get_user_data_interface()`)

| Method | Returns |
|---|---|
| `find_by_email_and_role(email, role)` | `UserORM \| None` |
| `find_by_id(user_id)` | `UserORM \| None` |
| `create_user(email, password, role)` | `UserORM \| None` |
| `list_all_users()` | `list[UserORM]` |
| `update_user_role(user_id, new_role)` | `UserORM \| None` |

Note: User repository methods return raw objects (not OperationResult) because they go through the legacy `safe_execute` whitelist adapter. Wrap in OperationResult at the class layer when needed.

**DatabaseInterface** (via `get_database_interface()`)

Used by `Admin` for backup/restore. Methods: `backup()`, `list_backups()`, `restore_backup(filename)`.

## safe_execute Whitelist

`SQLAlchemyDatabaseAdapter.safe_execute` only allows whitelisted action names:

```python
allowed_actions = {
    'find_user_by_email_role',
    'find_user_by_id',
    'create_user',
    'list_all_users',
    'update_user_role',
}
```

If you add new user operations, add the action name to this set AND implement the handler in `execute_query()`.
